template <class T> class auto_vector_ptr　【テンプレート】

☆　内容

auto_ptrExのvectorです。auto_ptrExは所有権インジケーターを持つ、ポインタで、デストラクトされるときに所有権を持っていれば、そのオブジェクトを自動的に解放します。これのvectorなので、管理しなければならないオブジェクトの数が動的に変化していく場合に対応できます。動的に変化しない場合は、auto_arrayを使います。

☆　メンバ関数

auto_ptrExのvectorです。vectorはSTLのものです。あらたに追加された関数は、次のものだけです。

void insert(T* pt){
　// 所有権を持ったauto_ptrの生成し、
　// auto_ptr間のコピーが発生し、所有権が移動する
　vector<auto_ptrEx<T> >::push_back(auto_ptrEx<T>(pt));
}

↑newしたポインタをvectorの最後に追加登録し、所有権を持たせる構文ですね。

void insert(auto_ptrEx<T> t){
// 所有権の移動を伴うauto_ptrExの追加登録。
vector<auto_ptrEx<T> >::push_back(t);
}

void insert(){
// あらたに生成して追加する。
vector<auto_ptrEx<T> >::push_back(auto_ptrEx<T>(new T));
}

☆　使いかた

すべてのオブジェクトを解放するには、vector::clear()を呼びます。これによりauto_ptrExのインスタンスが解放されるので、auto_ptrExが管理しているオブジェクトの所有権があるときは、auto_ptrExのデストラクタによってオブジェクトが自動的に解放されます。

配列を事前に確保するには、vector::resize()を呼びます。auto_ptrExはそのコンストラクタによってNULLに初期化されるので、こうやってサイズ変更を行なっても安全です。

オブジェクトを生成するには、auto_ptrEx::Add()を呼びます。具体的には、
　m_lpObject[123].Add(); 　　　　　　　　　　　　// 123番目のオブジェクトを生成する
　m_lpObject[123].Add(new COBject(x,y,z)); // 123番目のオブジェクトを生成する

生成したオブジェクトを、最後に追加するには、auto_vector_ptr::insert()を呼びます。具体的には、
　CObject* lp = new COBject;
　lp->Init(x,y,z);
　m_lpObject.insert(lp); // オブジェクトを追加する

だとか、
　auto_ptrEx<CObject> lp(new CObject);
　lp->Init(x,y,z);
　m_lpObject.insert(lp); // オブジェクトを追加する。所有権が移動するので、このあとlpがスコープアウトしても大丈夫！

また、クラスTのコンストラクタに引数が不要なのであれば、inertで引数を省略すれば、自動的にクラスTをnewして生成されたオブジェクトを、最後に追加します。
